Chapter 0: Design

Wireframing

Wireframing is like drawing blueprints for you house. You can plan logo placement, along with images, headings, and paragraphs.

Colour Theory

Colour theory tells you what colours invite or connect to what emotions. This will help you pick a colour scheme of two, maximum three colours to use for your website and visual brand. Colour theory resembles choosing paint colours for your house.

Typography

Typography is the pairing of fonts for the best visual presentation of your website, business card, and other promotional pieces that help your company stand out. Typography is like decorating your house, whether it be wallpaper or otherwise.

Whitespace

Whitespace gives elements breathing space. Whitespace resembles space between walls for living room, bathroom, bedroom, and any other room where you have to put furniture and appliances and also space between said furniture and appliances so there's room for you to move between them.

Chapter 1: HTML

Tags

HTML tags assist a browser in determining what to do with what's between the tags.

< !Doctype HTML >

!Doctype HTML is an instruction set for browsers, telling them how to interpret and render an HTML document. It prevents browsers from using an incompatible rendering mode, ensures backward compatibility, facilitates validation, and specifies document type and version.

< HTML >

HTML tags are your basic tags, telling your browser how to interpret and display what's between those tags. Without them, your other code like text etc. would not display properly.

< HEAD >

Head tags hold meta information, website title, and links to your stylesheet, JavaScript file(s), and other styling tidbits.

< TITLE >

Title tags denote a title, that goes in the browser at the top.

< BODY >

Body tags hold all the material that shows up in a website, like text and images.

< HEADER >

Semantic header tags denote what goes in the header space of a website, like logo, navigation, and anything else that should go in the header section.

< MAIN >

Semantic main tags tells the browser what goes in the main part of the website. This would include text and images.

< FOOTER >

Semantic footer tags tell the browser what goes in the footer section of a website. This would include a secondary navigaion, and anythig else that you would like to put there.

< SECTION >

Section tags denotes a section of a website.

< DIV >

Div tags denotes a block of content, for example an image with a description.

< ARTICLE >

Semantic article tags denote an article of interest.

< FIGURE >

Figure tags can wrap an image tag for better control of size.

< ASIDE >

Aside tags denote a sidebar, like in a blog where you can navigate to a certain blog post.

< META >

Meta tags go inside head tags, for example width=device width to tell the browser to prepare for responsiveness.

< LINK >

Link tags also go inside head tags, telling the html document where to look for css stylesheet to get its styles from, JavaScript files for running scripts, where to find styles for responsive typography fonts, and other things.

< NAV >

Nav tags denote navigation, which is where all your links to website sections and/or other pages go.

< UL >

Ul tags build an unordered list, with bullets next to each item.

< OL >

Ol tage build an ordered list, with numbers next to each item.

< LI >

Li tags denotes a list item either within an unordered or ordered list.

< H1 >

H1 tags denote the largest header tag for text. Normally goes in the header section.

< H2 >

H2 tags denote the next size of header text. Normally goes as a header on top of a section in the main section.

< H3 >

H3 tags denote the next size of header text. Not entirely sure where this one normally goes, but it would denote something important, I think.

< H4 >

H4 tags denote the next size of header text. Not entirely sure where this one normally goes, but it would denote something important, I think.

< H5 >

H5 tags denote the next size of header text. Not entirely sure where this one normally goes, but it would denote something important, I think.

< H6 >

H6 tags denote the next size of header text. Not entirely sure where this one normally goes, but it would denote something important, I think.

< P >

P tags denote a paragraph, so a block of text.

< STRONG >

Strong tags denote something of importance by making it bold.

< EM >

Em tags denote something of importance by making it italic.

< BR >

Br tags denote a page break in text, forcing any text after it to the next line.

< IMG />

Img tags enclose a path to an image to make it display on your web page.

< SPAN >

Span tags can isolate a word or block of text to make it a different colour, or some other style effect.

< A >

The a tag is used for links, like linking to a stylesheet. Example: a href="img/img.jpg"

Chapter 2: CSS

COLOR

The color selector changes your text to the color of your choice. (Note: I have a beef with this spelling as I am Canadian, and every time I have to type this selector out, I'm annoyed that it forces me not to spell it coloUr.

BACKGROUND

The background selector changes a background colour, or applies an image as a background.

WIDTH

The width selector sets a width for an image, or div (whether it be header, main, footer, section, etc.) to assist with layout, making it more conducive to site visitors staying on your site.

HEIGHT

The height selector sets a height for an image.

PADDING

The padding selector sets how much padding is between an element - say an image or text block - and it's outer container, for example a div.

MARGIN

The margin selector sets how much margin is between your div and the outer edge of the webpage... or between two divs.

FONT-FAMILY

Font-family sets what family the font comes from, so Open Sans is a sans-serif font. To properly call the font, you would set your selector as font-family { 'Open Sans', sans serif; }

FONT-SIZE

Font-size sets a size for your text. So, your h1 header could be 32px, and your paragraph text could be 16 px.

FONT-WEIGHT

Font weight sets your font weight. You can set it to bold, semi-bold, or normal.

LINE-HEIGHT

Line height sets a fixed height for a line of text.

TEXT-ALIGN

Text-align aligns your text to left, center, or right.

TEXT-DECORATION

Text-decoration sets your text as underline or not, depending on what you put in.

TEXT-TRANSFORM

Text-transform sets your text as all uppercase, all lowercase, or normal.

BORDER

Border sets a border around your divs, or text, or images, or anything else you want to put a border on. You can set border-left, border-right, border-top, border-bottom, or just border to have it go all the way around.

BORDER-RADIUS

Border radius sets your border radius, so 50% can make it a circular border. You can also set it anywhere in between 0 and 100 to make your border as round or sharp and pointy as you desire.

BOX-SIZING

Box sizing offers two options: content box gives you default css box sizing behaviour, which means if you set a box to 150px, the element itself will be 150px, plus the padding and margin you add to it. Border box, by contrast, accounts for element width plus any padding and margin added.

DISPLAY

Display selector offer the following options: block, inline-block, none, flex, and grid. Block gives you a, well, block, of text, inline block, which means they stack side by side, none - obviously no change, flex is part of flexbox, and grid (which I think is relatively new) makes elements display as a grid.

Chapter 3: Box


Model/Flexbox

FLOAT

Float allows you to move elements around in box model websites. You can float left or right.

CLEAR

Clear makes elements stack properly if float does not do the job.

PADDING

Padding is fairly important in box model as it determines how much space is between elements.

MARGIN

Margin is fairly important in box model as it determines how much space is between your elements and the outer edges of the browser.

ALIGN-ITEMS

Align-items is a flexbox property that makes elements align along a horizontal axis. You can align-items at flex-start, center, and flex-end.

JUSTIFY-CONTENT

Justify content is a flexbox property that makes elements align along a vertical axis. You can justify-content at flex-start, center, space-between, space-around and flex-end.

FLEX-DIRECTION

Flex-direction is a flexbox property for aligning and stacking elements.

FLEX-WRAP

Flex wrap is a flexbox property that, depending on which property you use, will either force items onto one line or wrap to multiple lines. Nowrap would mean it can wrap to multiple lines (to my thinking), wrap means wrap to one line... wrap reverse, I'm not sure.

FLEX-FLOW

Flex flow is a combination of flex-direction and flex-wrap, which gives you the options of row wrap, row wrap reverse, column wrap, and column wrap reverse.
